# DATA_CARD.md — данные решения

Провенанс данных, характер разметки, сплиты и слабый надзор для задачи
«Скажи мне, кто твой шлиф». Ключевая честность: **пиксельной экспертной GT нет** — есть метки
уровня папки, 42 примера синих линий и публичный proxy-датасет.

## 1. Официальный пакет Норникеля

- **Объём:** 1236 файлов, ~3.0 ГБ (`3 018 194 503` байт), манифест
  `dataset/_download_manifest.json` сверен `1236/1236`, SHA-256 проверен.
- **Изображения с метками уровня папки — 1180** (это **не** пиксельные маски, а сорт на аншлиф):
  - `рядовые` (обычные срастания) — **565**;
  - `тонкие / труднообогатимые` — **486**;
  - `оталькованные` — **129**.
- **Панорамы — 14** JPG, крупнейшая ≈ **27025 × 21227 px**, **без** разметки и класса
  (используем для стресс-теста производительности, не для метрик классификации).
- **«Области оталькования» — 42** изображения: тальк, обведённый **синей линией от руки**
  (примеры, а не сплошная разметка), + парные чистые оригиналы.
- **Формат:** OM (оптическая микроскопия) полированных шлифов; TIFF/PNG/JPEG.
- **Статус в репозитории:** `dataset/` — локальная копия, в git **не коммитится** (gitignored).

### Аудит дубликатов и конфликтов

SHA-256-аудит по размеченным папкам (`scripts/build_official_manifest.py`,
`official_label_audit`): **1180** размеченных изображений, **1124** уникальных хэша,
**56** групп дублирующегося контента, **24** группы с **конфликтующей меткой** (один и тот же
контент под разными сортами). Поэтому для честной оценки используется деконфликтованный сплит.

## 2. LumenStone S1/S2 (внешний proxy)

- Публичные полированные шлифы с **пиксельными масками минералов** — единственный реальный
  источник пиксельной разметки сульфидов.
- **S2** близок к Норильску (пирротин, халькопирит, пентландит, магнетит).
- Роль — **proxy-претрен** сегментации `сульфид/не-сульфид`, **не** официальная истина.
- Локально: `data/external/lumenstone/full/` (в git/Docker игнорируется); refresh —
  `scripts/download_lumenstone_all.sh`. Инвентарь внешних данных — [EXTERNAL_DATASETS.md](EXTERNAL_DATASETS.md).

## 3. Как из слабых источников собирается разметка

Пиксельной экспертной GT нет, поэтому сигнал собирается из независимых слабых источников, а
уверенность честно разделяется:

```text
LumenStone (реальные пиксельные маски)      -> претрен сульфид / не-сульфид
Otsu яркостно-морфологический на офиц. фото  -> псевдо-метки в домене задачи
зоны согласия источников                     -> обучающая метка
зоны разногласий и границы                   -> ignore (исключены из loss)
```

Разногласие источников **не** превращается в жёсткую метку — уходит в `ignore` и в QA. Тальк —
**отдельная** задача в не-сульфидной матрице.

### Производные обучающие наборы

| Набор | Состав | Назначение |
| --- | --- | --- |
| `binary_sulfide_dataset_v0` | 8536 тайлов 512 px (2976 LumenStone + 5560 офиц. псевдо-метка), шаг 384 | обучение сульфид-сегментации |
| `talc_non_sulfide_dataset_v0` | тайлы из 42 отревьюренных талька, только не-сульфид | обучение тальк-сегментации |
| grain dataset (путь B) | ~69k зёрен из 345 изображений (bbox + признаки формы) | grain-классификатор |

### Разметка талька (silver-GT)

42 blue-line изображения → выделение синего штриха по цвету → замыкание контура → заливка →
кандидат-маска → **пиксели самой разметки в `ignore`** (модель не учится на маркере) → перенос
на чистый парный оригинал → **ручное ревью всех 42** (`apps/talc_review_web.py`: кисть,
`Similar`, SAM2). Итог — silver-GT (не эксперт).

## 4. Сплиты для оценки

| Сплит | Состав | Файл | Назначение |
| --- | --- | --- | --- |
| **Деконфликтованный 345** ⭐ | 115/класс, sha256 conflict+dup исключены | `outputs/official_balanced_eval_split_deconflicted.json` | основная 3-классовая оценка (leak-free) |

Панорамы держатся **отдельно** как неразмеченные стресс-изображения (не входят в метрики
классификации).

## 5. Что честно значат эти данные

- **Нет пиксельной экспертной GT.** Сегментационные метрики — weak-label (сульфид) и silver
  (тальк); завышены относительно геологической точности.
- **Grade-GT — метка папки** на аншлиф, не пиксельная и не покомпонентная.
- **Эталонной доли талька не существует** (QA#4) — критерий «± 3%» не оценивается ни для кого;
  подробно в [EVALUATION.md](EVALUATION.md) §5.
- **Псевдо-метки — слабый надзор**, а не экспертная истина; мы это повторяем во всех метриках.
- **Данные не публикуются** в репозитории (официальный датасет — собственность организаторов;
  `dataset/` и `data/external/` в git не коммитятся).
