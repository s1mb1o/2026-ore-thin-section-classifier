# Component grade model: обученный классификатор компонентов ordinary/fine

Дата: 2026-07-04

Обновление 2026-07-05: интегрирован как новая opt-in ветка реализации в
текущий v2 пайплайн. Артефакт этой ветки —
`models/component_grade/hgb_weak100_nomag_20260705/model.joblib`: признаки
увеличения (`5x/10x/20x`) удалены, потому что первый артефакт выучил
смещение съемки, а не геологию. Порог компонента для класса `fine_intergrowth`
берется из `meta.json` (`component_threshold=0.65`).

## Что это

Обученная опциональная замена ручного OR-правила формы (`fine_dark_inside_ratio | fine_solidity_max |
fine_compactness_max`) в `component_analysis`: HistGradientBoosting по тем же
per-component признакам (+ log-area, заполнение bbox, аспект; без увеличения в
активном артефакте). Метки слабые — по папке класса (Рядовые → ordinary, Труднообогатимые →
fine). Классифицируются **найденные маски** (компоненты сульфидной сегментации), а
не кадр целиком, поэтому процентное соотношение ordinary/fine сохраняется — меняется
только присвоение метки компоненту перед той же площадной агрегацией.

## Цифры (бенчмарк: 100 тёмных+зелёных кадров ч1, тальк off)

| классификатор компонентов | точность кадра | ordinary recall | fine recall |
|---|---|---|---|
| правило OR (прод до этого) | 43.0% | 22.5% | 56.7% |
| первая модель `hgb_weak100_20260704` (с увеличением) | 73.0% | 57.5% | 83.3% |
| **активная модель `hgb_weak100_nomag_20260705`** | **66.0%** | **77.5%** | 58.3% |

In-sample первого финального артефакта на тех же 100: 80% (ожидаемо выше CV; честная оценка — 73%).
Активный артефакт выбран не по максимальной accuracy на этом узком сплите, а чтобы
убрать shortcut по увеличению и улучшить устойчивость рядового класса.
Известный системный кейс: магнетитовая плита с редкими сульфидными вкраплениями —
правило тянет в «рядовую» из-за компактности крупного куска; по вводной это
«тонкая» (см. обсуждение в сессии; таргет для следующей итерации разметки).

## Артефакт

Активный артефакт: `models/component_grade/hgb_weak100_nomag_20260705/{model.joblib,meta.json}`
(54 KB, обычный git-файл, не LFS). `meta.json` содержит список признаков, CV-метрики
и `component_threshold=0.65`.

Исторический артефакт сравнения: `models/component_grade/hgb_weak100_20260704/`.
Он содержит one-hot признаки увеличения и сохраняется только для воспроизводимости
первого эксперимента.
Обучение: `scripts/train_component_grade_model.py --runs-dir <batch>/runs --out-dir <dir>`.

## Интеграция (все точки — один параметр `component_classifier` в `analyze_components`)

- `src/ore_classifier/component_grade_model.py` — загрузка, признаки, labeler.
- `scripts/analyze_ore_from_masks.py --component-model <model.joblib>`
- `scripts/run_ore_pipeline.py --component-model …` (passthrough + поле в summary)
- `scripts/run_ore_pipeline.py --magnetite-prep` — опциональная двухпроходная
  подготовка для магнетитовых плит.
- `scripts/run_resident_batch.py --component-model … --magnetite-prep` →
  `ResidentSulfidePipeline(component_model=…, magnetite_prep=True)`.
- `apps/ore_pipeline_web.py --component-model … --magnetite-prep on` включает
  variant path. Без этих флагов web/CLI остаются на основном правиловом пути:
  `component_model=None`, `magnetite_prep=False`.

## Ограничения / дальше

- Обучено на слабых папочных метках 100 тёмных кадров ч1; ordinary recall остаётся
  главным резервом. Идёт ручное поклеточное ревью масок (`apps/grade_mask_review.py`,
  сид от этой же модели) — переобучение на честных метках должно поднять точность.
- Сегментация не отделяет магнетит от сульфидов; модель учится компенсировать это
  на уровне класса (магнетитовые плиты → fine), проценты минералогически неточны.
- Активный артефакт больше не использует увеличение из имени файла; разбор
  увеличения оставлен в коде только для совместимости с историческими данными и
  будущими явными вариантами.
