# Безмодельный эвристический классификатор оталькованных руд

Дата: 2026-07-04
Статус: зафиксированный боевой алгоритм (model-free baseline)

> ## ⚠️ ОБЯЗАТЕЛЬНО перед продакшеном: заменить ore-маску на модельную
>
> Шаг 1 алгоритма (ore/opaque-маска) в текущей реализации по умолчанию
> использует яркостный фолбэк `opaque_phase_mask`. **Его нужно заменить на маску
> из обученной сульфидной модели (SegFormer-B2, `models/binary_sulfide/...`).**
>
> Почему это критично: весь результат (matrix, порог, зоны, доля талька) считается
> от ore-маски. Фолбэк проверен только на 82 размеченных снимках (там IoU 97% с
> боевой `sulfide_mask_final`), но на свежих папках он завышал talcose. Точность
> 87.8% получена именно с качественной ore-маской.
>
> Как подключить: `detect_talc_zones(rgb, ore_mask=<маска из модели>)` или
> `classify_talcose_heuristic.py --ore-mask <sulfide_mask.png>`. Модельную маску
> берут из тайлового инференса `scripts/infer_binary_sulfide.py` (или из ore-
> пайплайна). Фолбэк оставить только как аварийный, когда модель недоступна.

Классификация OM-снимка полированного шлифа на **talcose / not-talcose** по
площади тальковых зон, без нейросети. Тальк — рассеянные тёмные хлопья в
нерудной матрице; talcose при доле зон талька > порога (≈ правило ТЗ «тальк
> 10%», но порог калибруется).

Код: `src/ore_classifier/talc_zone_heuristic.py`,
CLI: `scripts/classify_talcose_heuristic.py`,
регресс-тест: `tests/test_talc_zone_heuristic.py`.

## Алгоритм (пошагово)

```
RGB-снимок
1. ore/opaque-маска  — рудные/непрозрачные фазы (сульфиды + серый магнетит).
                       В продакшене — из обученной сульфидной модели; фолбэк —
                       яркостная opaque_phase_mask (мульти-доменный порог).
2. matrix            = весь кадр − ore.
3. dark threshold    = K × median(яркость matrix), K=0.85.
                       Тальк — самая тёмная фаза; K=0.85 ловит и светлый тальк,
                       0.752 недоловит, ≥0.95 — обвал (хватает силикат).
4. flakes            = тёмные связные компоненты МИНУС крупные СПЛОШНЫЕ пятна.
                       Крупное пятно выкидывается, только если оно непористое
                       (пористость = белые прожилки внутри залитого контура < 15%):
                       пора/тень/смола гладкая → убрать; плотный тальк «кружевной»
                       (с зазорами между чешуйками) → оставить.
5. zones             = proximity-агрегация хлопьев: морфологическое замыкание
                       радиусом Rc=25 (связать близкие хлопья, DBSCAN-подобно);
                       зона валидна, если внутри достаточно самих хлопьев
                       (площадь ≥ 0.03% кадра И заполнение ≥ 5%).
6. talc_fraction     = площадь зон / площадь matrix.
7. talcose если talc_fraction > 0.5 (калибруемый порог).
```

Все пространственные радиусы заданы при фиксированной ширине обработки
`proc_width=800`, чтобы результат не зависел от разрешения.

## Боевые параметры (утверждены 2026-07-04)

`TalcZoneConfig`: `k_threshold=0.85`, `blob_area_frac=0.01`,
`blob_porosity_max=0.15`, `footprint_close_radius=4`, `proximity_radius=25`,
`zone_min_flake_frac=0.0003`, `zone_min_flake_fill=0.05`,
`classify_threshold=0.50`, `proc_width=800`.

## Зафиксированный результат (82 размеченных снимка, sulfide_mask_final ore)

- **AUC (talcose vs остальные) = 0.809, точность = 87.8% (72/82)** при пороге
  зоны 53%. Точно воспроизводится регресс-тестом.
- IoU зоны с ручной разметкой (медиана talcose) ≈ 74%.
- Медиана доли зоны: TALCOSE 50% · ROW 22% · FINE 9%.

10 «ошибок» разобраны:
- **2 настоящих ложных срабатывания**: row_100 (разметка талька 3%), fine_138
  (0%) — тёмный силикат принят за тальк;
- **4 конфликта меток**: fine_48 (50% талька), row_-31 (44%), fine_109 (34%),
  row_1822217 (12%) — по разметке пользователя это talcose (правило >10%), а в
  папках рядовые/тонкие. Метод согласен с пиксельной разметкой; спорна папочная
  метка;
- **1 не-ошибка**: 2550381-1 (тальк 9% < 10%);
- **3 недоловли** светлого талька: t2m_-36 (72%), DSCN5032, 2550381-2.

Пользователь: расхождения — «ошибка классификации экспертом», метод ~корректен.

## Развитие эволюции метрики (как дошли до 0.809/0.85)

```
сырой порог (доля тёмного)              AUC 0.723
+ выкид сплошных пятен (пористость)     AUC 0.809 (flakes)
+ зоны по плотности (density D=0.12)    AUC 0.902  — но теряет разреженные хлопья
+ зоны по близости (proximity Rc=25)    AUC 0.809  — ловит разреженные хлопья, IoU↑
                                         ручная разметка = 0.940 (потолок)
```

Выбран **proximity Rc=25**: покрытие тальковых зон (IoU ~74% против 54% у
density) пользователь счёл «более корректным», ценой AUC (0.81 против 0.90).
Density-версия остаётся вариантом, если приоритет — чистая классификация.

## Потолок и что дальше

Яркость+морфология **не различают разреженный тальк и тёмный силикат**
(row_100/fine_138) — любой параметр, ловящий одно, ловит и другое. Разорвать
может только:
- **цветовой признак** (тальк vs силикат по оттенку) — быстрый эксперимент;
- **обученная модель** (SegFormer-B0 на разметке, IoU 0.644) — учит текстуру/цвет.

Важно: качество ore-маски критично. `opaque_phase_mask` (фолбэк) совпадает с
`sulfide_mask_final` на 82 (IoU 97%), но на свежих папках не проверен — в
продакшене ore-маску брать из сульфидной модели.

## Запуск

```bash
# один снимок (ore-маска из сульфидной модели предпочтительна)
python3 scripts/classify_talcose_heuristic.py \
  --image "data/Фото руд по сортам. ч1/Оталькованные руды/DSCN4718.JPG" \
  --out-dir outputs/talcose_heuristic_demo

# с готовой ore-маской
python3 scripts/classify_talcose_heuristic.py \
  --image path/to.jpg --ore-mask path/to_sulfide_mask.png --out-dir outputs/demo

# пакетно по папке
python3 scripts/classify_talcose_heuristic.py \
  --images-dir "data/Фото руд по сортам. ч2/оталькованные" \
  --out-dir outputs/talcose_heuristic_batch \
  --summary-csv outputs/talcose_heuristic_batch/summary.csv
```

Артефакты на образец: `talc_zone_mask.png`, `overlay.jpg`, `talcose_result.json`.
