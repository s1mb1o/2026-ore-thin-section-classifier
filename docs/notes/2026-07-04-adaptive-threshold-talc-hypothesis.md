# Проверка гипотезы: адаптивный яркостный порог для классификации оталькованных руд

Дата: 2026-07-04

## Гипотеза (предложена в ходе разметки)

1. Найти маску руды, исключить её из анализируемой области.
2. Порог = k × опорная яркость оставшейся матрицы (средняя или мода).
3. dark_fraction = доля тёмного в матрице; классификация: dark_fraction > T → оталькованная.
4. Подобрать (k, T) по 82 размеченным изображениям (62 оталькованных GT, 20 рядовых/тонких).

## Маска руды (артефакт, пригоден повторно)

Итеративно построена и **вручную верифицирована пользователем на всех 82 снимках**
(циклы просмотра контактных листов). Финал: `sulfide_mask_final.png` в
`outputs/talc_annotation_v1/samples/<id>/`. Структура — объединение доменных веток,
все пороги пер-снимочные:

- v2: абсолютная яркость `luma >= max(120, 0.75*p99.5)` — нормальные/яркие снимки;
- v3: относительная `>= 0.72*p99.5` + тёплый тон (R−B≥25) — тёмно-тёплые;
- v6: 2-ступенчатый Otsu (руда, затем серая фаза/магнетит) — холодные снимки
  (детект по медианной теплоте ярких пикселей < 40);
- v7/v8: 2-ступенчатый Otsu для ярко- и тёмно-тёплых (серая/бледная фаза);
- пер-снимочные overrides (`mask_overrides.json`): fine_75 thr=60, row_110 thr=50.
- Отклонено пользователем: density-заливка вкрапленных агрегатов (v10, откачена) —
  промежутки между зёрнами в маску не входят.

Маска = «непрозрачные/рудные фазы» (сульфиды + серая фаза), НЕ только сульфиды.
Медианы доли руды: talc_ch1 18%, row_ch2 29%, fine_ch2 14%, talcose_ch2 2.5%.

Попутный вывод: petroscope LumenStone-веса (ResUNet S1/S2) на официальных фото
непригодны (доменный сдвиг, кадр коллапсирует в один класс); MPS-бэкенд для той
архитектуры выдаёт мусор — только CPU.

## Результат: гипотеза НЕ подтверждается

Тривиальный базлайн (всё → оталькованная) = 75.6%.

| Вариант | лучший AUC | лучшая accuracy | corr с ручной долей талька |
| --- | ---: | ---: | ---: |
| A: k × средняя матрицы | 0.665 (k=0.6) | 0.622; 0.768 при вырожденном T | ≈ 0.02 |
| B: k × мода матрицы | 0.648 (k=0.8) | 0.793 (k=0.9) — едва выше базлайна | ≈ −0.04 |

Ошибки структурные, не шумовые:
- оталькованные ч1: dark_fraction 0.1–1.4% — порог k×ref на талькодоминантных
  снимках сидит ниже/внутри тальковой полосы (опора «матрицы» = сам тальк,
  циркулярность);
- рядовые/тонкие: до 21–27% dark_fraction — тёмные силикаты/тени считаются.

## Финальная проверка: ручная разметка порогов (82 кадра)

Пользователь вручную выставил «идеальный» порог бинаризации на каждом из 82
кадров через кнопку `Save threshold` в редакторе (порог + статистики матрицы
пишутся в `samples/<id>/manual_threshold.json`). Матрица = кадр − финальная
маска руды. Сбор: `scripts/collect_manual_thresholds.py`.

Что показала регрессия по этим 82 порогам:

1. **Порог хорошо ложится на статистику яркости**: `порог ≈ 0.75×median`
   (или ×mean), CV-MAE ≈ 5.9 luma, R² ≈ 0.64. Многофакторная модель (mean,
   median, mode, std, персентили) MAE не улучшает (5.90) — сигнал одномерный.
2. **При РУЧНОМ пороге разделение классов отличное**: AUC(dark_fraction,
   оталькованная-vs-остальные) = **0.946** — почти как у ручной доли талька
   (AUC 0.940).
3. **Но формульный порог разделение теряет** (формула не знает класс):
   - порог 0.752×median → AUC(dark_fr) = **0.727**;
   - порог 0.736×mean → 0.693;
   - честная LOO-регрессия → **0.662**.
   Медианы dark_fr по группам схлопываются: talcose 7.0% оказывается НИЖЕ
   talc_ch1 16.6% и row 12.0% — классы перемешаны.
4. **Механизм доказан напрямую**: остаток порога после формулы (median) кор-
   релирует с талькозностью **+0.66**. Пользователь добавлял +2.9 luma на
   оталькованных и −9.0 luma на рядовых/тонких сверх того, что даёт статистика,
   т.е. вносил ~12 luma классозависимой поправки — это его визуальное суждение,
   которого нет в глобальной яркости матрицы.

Причина (циркулярность, теперь доказана 4 способами): на талькодоминантном
кадре медиана/среднее матрицы = яркость самого талька, поэтому `k×median`
садится под тальковую полосу и ловит мало; на рядовом — выше и ловит больше.
Опора уезжает вместе с количеством талька.

Артефакты: `outputs/talc_annotation_v1/{manual_thresholds.csv,
manual_threshold_analysis.json}`, per-sample `manual_threshold.json`.

## Вывод

Глобальная яркостная статистика матрицы не измеряет тальк: опора коллапсирует на
талькодоминантных снимках, а межклассовое разделение ломается доменами съёмки.
Ручная разметка разделяет классы отлично (медианы 44–72% против 1.3%), значит
сигнал в данных есть, но он текстурно-контекстный. Путь для доли талька —
обученная модель (SegFormer-B0 talc folds: IoU 0.644/F1 0.782 на этой разметке);
яркостный dark_fraction годится максимум как слабый доп. признак.

Для презентации: «пороговую гипотезу проверили строго (82 GT, две опорные
статистики, ручная верификация масок руды) — не работает; поэтому тальк
детектируется обученной моделью».

Артефакты: `outputs/talc_annotation_v1/{sulfide_final_stats.json,
mask_overrides.json, experiment_final_items.json, sulfide_overlays_final/}`.
Скрипт масок — истории команд в сессии; при необходимости оформить в
`scripts/build_ore_opaque_mask.py` по этой заметке.
