# Протокол ручной разметки талька (v1)

Дата: 2026-07-03

Контекст: официальная blue-line разметка талька примерная (незамкнутые контуры,
частичное покрытие — медиана извлечённой доли 3.9% на «оталькованных» образцах).
Решение: собственная разметка с нуля + добор изображений из рядовых/тонких
классов, где тальк есть в малых количествах.

## Инструмент

Свой canvas-редактор `apps/talc_review_web.py` + scratch-режим:

```bash
# 1. Собрать воркспейс из произвольных изображений (можно мешать папки/классы)
python3 scripts/prepare_talc_scratch_workspace.py \
  --images "data/Фото руд по сортам. ч1/Оталькованные руды" \
  --images "data/Фото руд по сортам. ч1/Рядовые руды" \
  --images "data/Фото руд по сортам. ч1/Труднообогатимые руды" \
  --per-dir-limit 10 --shuffle-seed 7 \
  --output-dir outputs/talc_annotation_v1

# 2. Запустить редактор
python3 apps/talc_review_web.py --conversion-dir outputs/talc_annotation_v1
```

Инструменты: **Lasso** (обвёл от руки → залилось; Shift = стереть область),
Brush/Eraser, Polygon (точки, вставка на ребро, drag, ПКМ удалить), Rectangle,
опционально SAM2. Undo, autosave, `Save` / `Save and next`.
Результат на образец: `reviewed/reviewed_talc_mask.png`,
`reviewed_ignore_mask.png`, `reviewed_overlay.png`, `review_patch.json`.

## Что считать «долей талька» (конвенция v1)

Официальные источники: docx — «присутствует более 10% талька. На изображениях
тальк может выражаться в виде тёмных рассеянных областей в нерудных зонах»;
страница ТЗ — «ошибка оценки доли талька ≤ ±3% относительно экспертной
разметки». Эталон — обведённые экспертом ЗОНЫ, поэтому:

- **официальное число**: `talc_zone_fraction` = площадь зон оталькования /
  анализируемая площадь (минус артефакты/рамки); сульфидные зёрна внутри зоны
  вычитаются (тальк — фаза в нерудной матрице); правило `>10%` применяется к
  этому числу;
- **доп. метрика**: `talc_density_in_zones` = доля тёмных тальк-пикселей внутри
  зон — фиксирует «плотно/разреженно», в правиле не участвует;
- плотность зоны НЕ влияет на официальную долю: плотная и разреженная зоны
  равной площади дают равный вклад.

**Конвенция разметки (v2, пересмотрена на реальном примере): размечаем
локальные пятна-агрегаты, зону выводим программно.** Экспертные зоны содержат
крупные чистые окна без тальковой текстуры; помечать их позитивом — давать
модели противоречивый сигнал (одинаковая гладкая текстура = тальк внутри зоны
и не тальк снаружи). Поэтому:

- пиксельный слой (наша разметка → обучение): обводим агрегаты тальковых пятен
  плотно; чистые окна внутри бывших «зон» — негатив;
- зернистость: агрегат = группа пятен с матрицей между ними на масштабе самих
  пятен; чистое окно вырезаем, только если оно заметно крупнее типичного
  расстояния между пятнами (ориентир >~100–150 px при 2272×1704); отдельные
  пятнышки по одному не обводим;
- зонный слой (официальное число): зона = дилатация/слияние близких пятен с
  радиусом R; R калибруется по 31 пригодной экспертной blue-line маске так,
  чтобы выведенные зоны воспроизводили площади экспертных зон; R логируется;
- направление «пятна → зона» всегда выводимо, обратное — нет; поэтому плотная
  разметка информационно строго богаче зонной;
- сульфидные зёрна внутри агрегата выгрызать не нужно — вычитаются по
  сульфидной маске автоматически.

**Предобработка и разметка:** размечаем исходные изображения (модель учится на
сырых пикселях; запечённая предобработка = train/inference skew и раздвоение
провенанса). Для тёмных/зеленоватых снимков в редакторе есть display-only
режим `View: Boost dark / Boost dark ++` — усиливает яркость/контраст только
на экране, маски сохраняются в координатах исходного изображения.

Известная нестыковка: извлечённые из blue-line зоны на «оталькованных» снимках
дают медиану ~3.9% (<10%). Вероятные причины: частичная/примерная разметка,
незамкнутые контуры (9/42 дали 0%), и то, что «>10%» — свойство пробы, а не
каждого поля зрения. После собственной переразметки порог правила калибруется
по папочным меткам (`scripts/calibrate_ore_rules.py`), буквальные 10% не
считаются догмой; при стабильном расхождении — вопрос организаторам.

## Почему тальк тёмный в отражённом свете (обоснование прокси)

По минералогическим данным (Mindat, тальк Mg3Si4O10(OH)2):

- твёрдость 1 по Моосу (самый мягкий минерал) → в полированном шлифе
  выкрашивается/размазывается, даёт рельеф → рассеивает свет → темнее соседей;
- показатель преломления n≈1.54–1.60 → отражательная способность ~5% против
  пирротина ~35–40%, халькопирита ~44%, пентландита ~45%, магнетита ~20% →
  иерархия яркости: сульфиды >> магнетит > силикаты >= тальк;
- типичные ассоциации талька: пирит, магнетит, серпентин, карбонаты; генезис —
  изменение мафит-ультрамафитов (норильский контекст, тальк вокруг сульфидов);
- ограничение: серпентин/хлорит и др. тёмные силикаты оптически почти
  неотличимы от талька → задача официально про прокси «тёмная рассеянная
  фаза» (docx: «будем ориентироваться на них»), отсюда необходимость
  hard-negatives (тени, смола, поры, царапины — не тальк);
- морфология (петрографические справочники, PPL habit): тальк — листоватые/
  радиально-лучистые агрегаты мятых чешуек и волокон, спайные листочки
  изогнуты → на аншлифе зона талька имеет рыхлую чешуйчатую текстуру с
  нечёткими перистыми краями. Отличия от ложных срабатываний: поры/смола —
  бесструктурно-чёрные с резкими краями; царапины — прямые линии; краевые
  тени — плавный градиент без текстуры.

## Правила разметки

1. **Исчерпывающая разметка на изображении**: размечаем ВЕСЬ видимый тальк на
   каждом взятом изображении. Тогда неразмеченное на нём — честный негатив,
   и по изображению можно мерить ошибку доли (бар ТЗ ±3 п.п.).
2. **Три категории**: тальк (уверенно) → маска; сомнительное/кайма границ →
   ignore; остальное — негатив автоматически.
3. **Границы не вылизывать**: рассеянная фаза без точной границы; важна
   консистентность решений между изображениями, а не пиксельная точность.
4. **Стратификация выборки**: все визуальные домены (зеленоватые/тёмные/светлые
   снимки), ч1 и ч2, разные увеличения (5x/10x/20x), и обязательно изображения
   из рядовых/тонких (мало талька → hard negatives против «всё тёмное = тальк»).
5. **Размечаем сырые изображения** — без предобработки; вариативность цвета
   закрывается аугментацией при обучении.
6. **Итеративно**: после первых ~10 изображений обучить быстрый черновой
   классификатор → предзаполнять маску на следующих → исправлять, а не рисовать
   с нуля.

## Целевой объём (time-boxed)

- ~20–30 изображений: ~15 оталькованных + ~10 рядовых/тонких.
- Из них 8–10 — отложенный eval, разбиение по стемам образцов
  (`2550375-1`/`-3` — один физический образец, не разделять train/val).

## Препроцессинг входа модели (ablation, решается экспериментом)

Тальковая модель ещё не обучена → выбор входного представления открыт (это НЕ
случай замороженной сульфидной модели, где новый препроцессинг = skew). При
30–40 изображениях снижение входной вариативности экономит данные; признак
талька относительный («темнее локального окружения»), а абсолютная яркость
гуляет между доменами съёмки — нормализация может помочь.

Варианты (вход train == inference, детерминированно, логируется):

- A (база): сырой RGB + сильная фотометрическая аугментация;
- B: flat-field (деление на гауссово размытие) + gray-world нормализация;
- C: B + CLAHE по L (риск: шум/гало на краях сульфидов).

Выбор по MAE доли талька на отложенных размеченных изображениях; победитель
фиксируется в конфиге и карточке модели. Разметка от этого не зависит:
преобразования фотометрические, маски привязаны к пиксельной сетке и валидны
для всех вариантов.

## Модель после разметки

SegFormer-B0 (ImageNet-init `nvidia/mit-b0`), существующий тренировочный путь
с ignore-масками; выбор чекпоинта по MAE доли талька на отложенных полностью
размеченных изображениях, не по tile-IoU. Параллельно scribble-RF как второй
источник и помощник разметки. Инференс талька — только вне сульфидной маски.
