# Метрики проекта: классификация руды по шлифам

Дата: 2026-07-04. Ветка: `component-grade-model`.

Пайплайн выносит кадровый вердикт в три шага: **(1)** сегментация рудной
(непрозрачной) фазы — SegFormer-B2; **(2)** классификация каждого найденного
рудного компонента на «обычное/тонкое срастание» — обученная модель компонентов
(`models/component_grade/hgb_weak100_20260704`); **(3)** талько-ветка — тальковые
зоны в нерудной матрице; при доле талька выше порога кадр уходит в
«оталькованная» раньше, чем решается рядовая/тонкая. Ниже метрики двух решающих
веток на официальных данных организатора. Обе оценки — **кадровый вердикт всего
пайплайна**, а не изолированного классификатора: сегментация B2 и площадная
агрегация участвуют в каждом предсказании.

---

## 1. Рядовая ↔ труднообогатимая (модель компонентов)

**Бенчмарк**: 100 кадров «Фото руд по сортам. ч1» (40 Рядовые / 60
Труднообогатимые), отобранные как самые тёмные из зелёнодоминантных — сложный
сегмент, на котором прежняя логика работала хуже случайного. Оталькованные
исключены, талько-ветка отключена. Метки — папка класса организатора.

**Честность оценки**: image-level 5-fold GroupKFold — каждый кадр предсказан
моделью, не видевшей его при обучении; компоненты одного кадра никогда не
разделяются между train/test. Повторные разбиения дают 73–74% (±1 кадр).

| метрика (кадровый вердикт) | значение |
|---|---:|
| **Точность (accuracy)** | **74.0%** |
| Macro-F1 | 72.1% |
| F1 тонкая / рядовая | 79.4% / 64.9% |
| Recall тонкая / рядовая | 83.3% / 60.0% |
| Precision тонкая / рядовая | 75.8% / 70.6% |
| AUC (площадная fine-доля как скор) | 0.762 |

Confusion (GT → предсказание): рядовая→рядовая 24, рядовая→тонкая 16,
тонкая→рядовая 10, тонкая→тонкая 50.

Контекст: тривиальный бейзлайн «всегда тонкая» на этом наборе даёт 60%.
Модель классифицирует **найденные маски** (компоненты), поэтому процентное
соотношение срастаний в отчёте пайплайна сохраняется. Известный остаточный
класс ошибок — гигантские сросшиеся компоненты (один кусок ≈ вся руда кадра)
с одной меткой на всю массу; ведётся поклеточная переразметка
(`apps/grade_mask_review.py`) для переобучения на честных метках.

## 2. Оталькованная ↔ не оталькованная (тальковые зоны)

**Бенчмарк**: 82 вручную размеченных кадра (плотная пиксельная разметка талька,
non-expert): 42 «ч1/Оталькованные руды», 20 «ч2/оталькованные», 10 «ч2/рядовые»,
10 «ч2/тонкие». GT кадра — папка организатора (talcose vs не-talcose).

Алгоритм модельно-независимый (яркость+морфология): матрица = кадр − рудная
маска; бинаризация 0.85×медианы яркости матрицы; фильтр сплошных пятен по
пористости; объединение хлопьев в зоны (радиус 25); вердикт по доле зон.

| конфигурация рудной маски | AUC | точность | порог |
|---|---:|---:|---:|
| **SegFormer-B2 (прод)** | **0.817** | **89.0%** | 0.37 |
| яркостная эвристика (фолбэк) | 0.809 | 87.8% | ~0.50 |

Разбор 10 ошибок конфигурации 0.809/87.8: 4 — конфликты папочной метки (наша
разметка нашла 12–50% талька на кадрах из «рядовых/тонких» → по правилу >10%
должны быть talcose), 2 — тёмный силикат, принятый за тальк, 3 — пропущенный
светлый тальк, 1 — тальк <10% в «оталькованной» папке. Т.е. значимая часть
«ошибок» — шум папочных меток, не алгоритма.

### 3.2 Тальк (AUC 0.817 / 89.0% @ 0.37)

Файлы: 82 кадра из `datasets/talc_annotation_v1/manifest.csv` (ветка
`talc-annotation-tools`; манифест содержит `source_relpath` + `source_sha256`
для однозначного сопоставления с набором организатора; там же — маски нашей
разметки талька).

```bash
# 1) рудная маска B2 для каждого кадра
python3 scripts/infer_binary_sulfide.py --image <кадр> \
  --checkpoint models/binary_sulfide/segformer_b2_dataset_v0_zelda_20260703_overnight_safetensors/best.pt \
  --out-dir <out>/<кадр>

# 2) талько-вердикт (скрипт на ветке talc-annotation-tools)
python3 scripts/classify_talcose_heuristic.py --image <кадр> \
  --ore-mask <out>/<кадр>/sulfide_mask.png \
  --classify-threshold 0.37 --out-dir <talc-out>
```

AUC/точность считаются по `talc_fraction` из `talcose_result.json` против GT
«кадр лежит в папке оталькованных». Вариант с фолбэк-маской — те же команды без
`--ore-mask` (порог ~0.50). Полный алгоритм и калибровка:
`docs/notes/2026-07-04-heuristic-talcose-classifier.md` (та же ветка).

## 4. Сводка одним абзацем (для презентации)

На официальных данных организатора пайплайн даёт: **тальк — AUC 0.82,
точность 89%** (82 пиксельно размеченных кадра, прод-маска B2, порог 0.37);
**рядовая/тонкая — точность 74%, macro-F1 0.72** (100 сложных тёмных кадров ч1,
честная image-level кросс-валидация; прежняя эвристика формы давала 43% — ниже
случайного). Обе ветки классифицируют найденные маски, а не кадр целиком,
поэтому вместе с вердиктом пайплайн выдаёт процентные доли срастаний и талька.
