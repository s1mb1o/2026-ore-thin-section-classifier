## Слайд «Сложность задачи»

**ЗАГОЛОВОК:** Сложность задачи

**Саблайн:** Каждая проблема постановки → наше инженерное решение

| Проблема | Как мы решили |
| --- | --- |
| Субъективность: специалисты по-разному читают характер срастаний | Детерминированное правило из ТЗ + измеримая морфология каждого зерна |
| Нет пиксельной экспертной разметки — только метки папок и 42 фото, где зоны талька лишь приблизительно обведены синей линией от руки | Собираем обучающую разметку из слабых источников: претрен на публичном LumenStone (настоящие пиксельные маски) + автоматические псевдо-метки на официальных фото. Модель учится только там, где источники согласны; спорные пиксели исключаются из обучения |
| «Обычные» и «тонкие» срастания различаются строением зерна — степенью и характером его замещения нерудной фазой, — а не яркостью или цветом пикселей | Классификация на уровне зерна по морфологии: доля замещения нерудной фазой, сплошность (solidity), компактность формы |
| Гигапиксельные панорамы до 27025×21227 px | Tiled-инференс с перекрытием и сшивкой — без ресайза, морфология сохранена |
| Тальк — самый редкий и хуже всего размеченный класс: 129 фото из 1180, а зоны талька обведены лишь на 42 | Отдельный детектор талька + собственный инструмент ревью масок |

## Слайд «Наш пайплайн»

**ЗАГОЛОВОК:** Наш пайплайн

**Саблайн:** Не чёрный ящик, а проверяемый измерительный контур — каждый этап виден и проверяем отдельно

```text
панорамное OM-изображение
  → бинарная сегментация: сульфид / не сульфид (SegFormer-B2)
  → детектор талька в нерудной матрице (SegFormer-B0)
  → связные компоненты — отдельные зёрна
  → обычные / тонкие срастания по морфологии зерна
  → детерминированное правило класса руды (из ТЗ)
  → маска + проценты + confidence + отчёт (PDF/CSV)
```

**Акцент:** Мы шли **снизу вверх** — начали с самого надёжно поставленного и лучше
всего размеченного вопроса («где сульфидные пиксели?») и только на проверенном
фундаменте строили следующие этажи: зёрна → тип срастания → тальк → сорт руды.
Это даёт устойчивость: ошибка не размазывается по чёрному ящику, а локализуется
на своём этапе, где её видно и можно исправить.

**Инженерное решение:** по правилу ТЗ, если тальк > 10%, руда — оталькованная, и
считать обычные/тонкие срастания уже не обязательно. Но мы **сознательно доводим
конвейер до конца** на каждом запуске: оператор видит полную картину, может
скорректировать результат — и эта коррекция становится обратной связью и данными
для дообучения моделей.

**Внизу слайда:** Геолог проверяет не только итог, но и причину итога: где рудная
фаза, какое срастание у конкретного зерна, где тальк, как посчитан процент.
LLM в контуре решения нет.

## Слайд «Сильные стороны нашего пайплайна»

**ЗАГОЛОВОК:** Сильные стороны нашего пайплайна

**Саблайн:** Интерпретируемость + инженерная зрелость

- **Проверяемость каждого этапа** — любой процент прослеживается до маски, источника сигнала и параметров запуска (immutable-запуски, провенанс);
- **Классификация на уровне зерна**, а не ярлык на весь кадр: система показывает, какое именно зерно и почему повлияло на сорт;
- **Сильная сульфидная сегментация** — IoU 0.974 (weak-label), лучший из 4 бенчмарк-вариантов;
- **Продукт, а не демо-скрипт**: вьюер со слоями, правка маски и пересчёт, история, Серии, PDF/CSV/ZIP, REST API, Docker;
- **Два бэкенда**: ML (SegFormer) и эвристика без torch — fallback и объяснимый baseline;
- **Честные метрики**: разделены по источнику (weak-label / silver / proxy), спорные пиксели в ignore, а не в метку.

## Слайд «Слабости нашего пайплайна»

**ЗАГОЛОВОК:** Слабости нашего пайплайна

**Саблайн:** Мы знаем свои разрывы до цели — и они измеримы

- **Нет экспертной пиксельной GT** — все сегментационные метрики weak-label/silver, числа завышены относительно «геологической» точности;
- **Ошибка доли талька ≤ ±3 п.п. измерена и не выполняется** — leak-free OOF на 42 silver-масках: медиана 5.53 п.п., только 33% в пределах ±3 п.п.; критерий не сертифицируем в принципе — эталонной доли талька нет ни у нас, ни у жюри, ни у организаторов (QA#4);
- **Тальк — самый хрупкий источник**: 42 фото с общими условиями съёмки, silver-маски с частичной цикличностью;
- **Детерминированное правило слабо на оси обычное↔тонкое** (macro-F1 0.21/0.46) — критерий ТЗ «≥ 90%» закрыт отдельной моделью (Grade-CNN, 0.930/0.939), а не правилом;
- **Эвристика пре-лейбла путает рваный край с тонким срастанием** — поэтому запущена ручная разметка зёрен и обучение grain-классификатора (путь B).

## Слайд «Обучение: сульфид-сегментация»
Лимит 82×20: тело слайда — 18 строк.

**ЗАГОЛОВОК:** Обучение: сегментация сульфидов (SegFormer-B2)

**Саблайн:** Слабый надзор из независимых источников — разногласия не в метку, а в ignore

```text
Разметка (пиксельной экспертной GT нет — собираем сигнал сами):
   LumenStone S1/S2, реальные пиксельные маски → претрен «сульфид/не сульфид»
   официальные фото: сульфиды светлее фона → авторазметка порогом
      яркости (метод Оцу) = псевдо-метки прямо на данных задачи
   зоны согласия источников → обучающая метка
   зоны разногласий и границы → ignore: не штрафуются и не поощряются

Модель и обучение:
   SegFormer-B2, backbone предобучен на ImageNet (nvidia/mit-b2)
   8536 тайлов 512 px (2976 LumenStone + 5560 официальных), шаг 384
   CrossEntropyLoss(ignore_index) — спорные пиксели вне функции потерь
   аугментации: флипы, повороты 90°, мягкий color-jitter

Результат (против слабой разметки, не экспертной GT):
   Val IoU 0.974 · F1 0.987 · AUC 0.999 · HD95 23.6 px
   B2 победил в бенчмарке 6 архитектур:
   ResUNet 0.956 · B0 0.953 · B1 0.972 · B3 0.973 · Mask2Former 0.968
```

## Слайд «Обучение: talc-сегментация»

**ЗАГОЛОВОК:** Обучение: сегментация талька

**Саблайн:** Из 42 фото с синими линиями от руки — в обученный детектор

```text
42 фото «Области оталькования» (синие линии от руки)
  → выделение синего штриха → замыкание контура → заливка → кандидат-маска
  → пиксели самой разметки в ignore (модель не учится на маркере)
  → перенос маски на чистый парный оригинал
  → ручное ревью всех 42 масок (talc_review_web: кисть, Similar, SAM2) → silver-GT
  → обучение SegFormer-B0 только в нерудной матрице, 5-fold
```

- **IoU 0.644 / F1 0.782** (5-fold, против silver-масок);
- побит oracle-luma baseline (0.502); старый HSV-кандидат давал IoU 0.000;
- тальк ищется только в матрице: анализируемая область − сульфиды − артефакты;
- честно: silver-маски — не экспертная GT, все 42 фото делят условия съёмки.

Если остаётся место (или переносом на слайд с пожеланиями ТЗ):

```text
Сверх функциональных требований (пожелания ТЗ)
   Обратная связь для дообучения: правка маски → новый запуск,
      разметка зёрен/талька → датасеты («экспертная проверка»)
   Метаданные: масштаб, условия съёмки, месторождение
   ГИС-экспорт: GeoJSON и Shapefile · Руководство пользователя
   Локальное развёртывание: Docker, без внешних сервисов
   Крупнейшая панорама датасета 27025×21227 (574 Мп) — end-to-end за
      7 мин 08 с на RTX 4090, пик GPU-памяти 4777 MiB, ~4.7 ГБ (замер 2026-07-05)
```

Замеренные факты по панорамам (docs/benchmarks/07, 08):

- цель ТЗ ≤ 5 мин на 10000×10000 закрыта: ближайшая панорама 13330×9489
  (126 Мп) — 186.9 с на Mac M2 Max, 99.6 с на gx10, 104.3 с на zelda;
- крупнейшая панорама 27025×21227 (574 Мп) — 427.9 с end-to-end (ML
  сульфиды + ML тальк), т.е. ~4 раза быстрее целевого темпа ТЗ;
- пик GPU-памяти 4777 MiB, ~4.7 ГБ — не растёт с размером панорамы (тайлы + memmap);
  честная оговорка: пиковая системная RAM ~20 ГБ, нужна станция с 32 ГБ.

## Слайд «Как запустить»

**ЗАГОЛОВОК:** Как запустить

**Саблайн:** Три способа — от готового облака до одной локальной команды

**1. Развёрнутое решение (ничего ставить не нужно):**
`https://nornickel-ai-hackathon.alola.ru` — Selectel, 1×L4 24 ГБ
(резерв: `https://nornickel-ai-hackathon.my.3simbio.ru/workspace`, DGX Spark)
доступ: reviewer / пароль на слайде «Ссылки»

**2. Docker (одна команда):**
```bash
docker compose -f docker-compose.ore-pipeline-ui.yml up --build
# → http://<host>:8080/workspace
```

**3. Локально, без Docker:**
```bash
python3 apps/ore_pipeline_web.py --host 127.0.0.1 --port 0
# ML-бэкенд: --backend ml --checkpoint models/binary_sulfide/.../best.pt
```

- работает без GPU (heuristic-бэкенд без torch) и с GPU (NVIDIA / Apple MPS);
- локальный запуск — без внешних сервисов, для конфиденциальных данных.
