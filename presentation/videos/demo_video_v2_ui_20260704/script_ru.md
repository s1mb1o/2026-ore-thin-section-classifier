# Сценарий озвучки v2 UI demo

Голос: ru-RU-DmitryNeural, rate -12%. Итоговая длительность видео: 5:00.

## 01. Постановка задачи [0.00-5.93s]

Это демонстрация v2 интерфейса решения «Скажи мне, кто твой шлиф».

## 02. Постановка задачи [5.93-12.98s]

Задача простая для пользователя и сложная для модели: по одному оптическому изображению определить сорт руды.

## 03. Пример шлифа [12.98-20.31s]

Мы различаем обычные сростки, труднообогатимые тонкие сросты и оталькованные руды, где важна доля талька.

## 04. Что видит модель [20.31-28.87s]

На экране виден реальный run из истории v2 UI: исходный шлиф, маски и результат уже сохранены как неизменяемые артефакты.

## 05. Данные без утечек [28.87-36.10s]

Сначала мы проверяем данные, потому что официальные папки размечены на уровне изображения, а не пиксельными масками.

## 06. Аудит датасета [36.10-42.72s]

Аудит нашёл пятьдесят шесть групп дубликатов по содержимому и двадцать четыре группы с конфликтующими метками.

## 07. Честная оценка [42.72-51.47s]

Такие изображения исключены из оценки, а split делается по образцу, чтобы одинаковый шлиф не попадал одновременно в обучение и проверку.

## 08. Оговорка по разметке [51.47-59.63s]

Поэтому все метрики в ролике трактуются аккуратно: weak supervision помогает учиться, но не заменяет экспертную геологическую истину.

## 09. Пайплайн [59.63-66.90s]

Пайплайн начинается с бинарной сегментации сульфидов, затем считает связные компоненты и признаки каждого зерна.

## 10. Пайплайн [66.90-75.36s]

После этого отдельная ветка оценивает тальк, а финальный сорт получается из долей, морфологии и явных правил принятия решения.

## 11. Метрики сегментации [75.36-87.05s]

Для сульфидов текущий SegFormer-B2 даёт IoU ноль целых девятьсот семьдесят четыре тысячных, F1 ноль целых девятьсот восемьдесят семь тысячных и AUC ноль целых девятьсот девяносто девять тысячных.

## 12. Метрики классификации [87.05-98.12s]

Классификация сорта на deconflicted split из трёхсот сорока пяти изображений даёт macro F1 ноль целых семьсот сорок четыре тысячных и macro AUC ноль целых восемьсот восемьдесят тысячных.

## 13. История запусков [98.12-106.28s]

Теперь переходим в приложение: оно хранит историю запусков, и каждый run можно загрузить, не пересчитывая тяжёлый ML заново.

## 14. Одиночный шлиф [106.28-112.98s]

В рабочем месте оператор видит загруженный шлиф, выбранную предобработку и доступные слои результата.

## 15. Финальная маска [112.98-121.61s]

В финальном слое зелёный цвет показывает обычные сростания, красный — тонкие, синий — тальк, а фон остаётся отдельным классом.

## 16. Сульфидный слой [121.61-129.30s]

Переключаемся на слой сульфидов: можно менять прозрачность, включать контуры и быстро сверять маску с исходным изображением.

## 17. Артефакты подготовки [129.30-137.48s]

Кнопка «Исправить» открывает ручную разметку артефактов шлифовки; отмеченные области исключаются из знаменателя метрик.

## 18. Результат [137.48-146.39s]

Ниже приложение пишет человекочитаемый вывод: сорт, долю сульфидов, долю талька и объяснение, почему сработало именно это правило.

## 19. Таблица метрик [146.39-154.65s]

Таблица метрик показывает проанализированную область, общую долю сульфидов, обычные и тонкие срастания, тальк и артефакты.

## 20. Компоненты [154.65-163.30s]

Если в метаданных задан калиброванный масштаб, в CSV и отчёте дополнительно появляются физические площади в квадратных микрометрах и миллиметрах.

## 21. Зерновой отчёт [163.30-172.31s]

Отдельная таблица сульфидных зёрен показывает площадь, эквивалентный диаметр, периметр, долю сульфидов и proxy-показатели освобождения.

## 22. Контроль зерен [172.31-179.88s]

Выбранные строки можно подсветить на изображении, чтобы эксперт быстро проверил, какие компоненты влияют на решение.

## 23. Экспорт [179.88-189.52s]

Все файлы запуска открываются из встроенного списка: изображения, CSV, JSON, PDF-отчёт и ZIP-архив остаются связаны с тем же run id.

## 24. Ревью талька [189.52-198.69s]

Тальк показываем отдельно, через приложение ревью масок, потому что исходные синие контуры — это примерные области, а не полная пиксельная истина.

## 25. Проверенные маски [198.69-205.76s]

Здесь видны positive bag, подтверждённый talc node, защита от сульфидов и статус reviewed для образца.

## 26. Метрики талька [205.76-216.19s]

После ручного ревью сорока двух образцов талковая SegFormer-B0 ветка даёт mean IoU ноль целых шестьсот сорок четыре тысячных и F1 ноль целых семьсот восемьдесят две тысячных.

## 27. Оговорка по тальку [216.19-225.34s]

Важная оговорка: примерно восемьдесят три процента проверенного талька оказалось вне исходных синих мешков, поэтому мы не завышаем точность доли талька.

## 28. Панорама [225.34-232.32s]

Для больших панорам v2 UI использует анализ в уменьшенном представлении и тайловую обработку с перекрытием.

## 29. Тайлы [232.32-239.92s]

На кадре включена сетка тайлов: так видно, как большая панорама разбивается на участки без ручного кадрирования пользователем.

## 30. Панорамный результат [239.92-249.45s]

Панорамный run также сохраняет сорт, доли классов, маски и отчётные файлы, но долгий расчёт для демо открывается из истории и не тратит минуты в кадре.

## 31. Статус системы [249.45-258.77s]

Страница статуса показывает активный backend, checkpoint, состояние CPU, памяти, диска, историю запусков и предупреждения по среде.

## 32. REST API [258.77-267.96s]

Для интеграции есть REST API: загрузка изображений, запуск, опрос статуса, артефакты, настройки, серии и health checks.

## 33. Воспроизводимость [267.96-278.59s]

Решение разворачивается локально и офлайн через Docker для x86 и ARM64; обычное изображение стартует примерно за восемнадцать целых шесть десятых секунды.

## 34. Финальный вывод [278.59-289.91s]

Главная идея v2 UI — не просто показать красивую маску, а сохранить воспроизводимый пакет: исходник, настройки, модели, метрики, отчёты и ограничения.

## 35. Финальный вывод [289.91-299.00s]

Следующий инженерный шаг — экспертная валидация масок талька и калибровка правила сорта, но текущий pipeline уже готов для понятного судейского демо.
